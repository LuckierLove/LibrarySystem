# 系统使用说明

## 一、系统概述

图书借阅管理系统是一个基于Java Swing开发的桌面应用程序，主要用于管理图书的借阅和归还。系统分为管理员和读者两种角色，提供了完善的图书管理和借阅管理功能。

## 二、系统功能模块

### 2.1 登录模块

**功能描述**：
- 用户通过用户名和密码登录系统
- 系统验证用户身份和账号状态
- 登录成功后进入主界面

**操作步骤**：
1. 启动应用程序，显示登录界面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 系统验证通过后进入主界面

**界面说明**：
- 标题：图书借阅管理系统
- 输入框：用户名、密码
- 按钮：登录、重置

### 2.2 图书浏览模块

**功能描述**：
- 以表格形式展示所有图书信息
- 显示图书ID、书名、作者、出版社、分类、总数量、可借数量、价格
- 支持关键词搜索功能

**操作步骤**：
1. 登录后自动加载所有图书
2. 可以通过搜索框输入关键词搜索
3. 点击"刷新"按钮可重新加载图书列表

### 2.3 借书模块

**功能描述**：
- 用户可以借阅有库存的图书
- 系统自动检查库存和重复借阅
- 支持自定义借阅天数

**操作步骤**：
1. 在图书列表中选择要借阅的图书（单击某一行）
2. 点击"借书"按钮
3. 在弹出的对话框中输入借阅天数（建议30天）
4. 系统验证库存并创建借阅记录
5. 借书成功后，图书可借数量自动减1

**业务规则**：
- 图书库存必须大于0才能借阅
- 用户不能重复借阅同一本书
- 借阅天数限制在1-365天之间
- 系统自动计算应还日期

### 2.4 还书模块

**功能描述**：
- 用户可以归还已借阅的图书
- 系统自动更新借阅记录和图书库存

**操作步骤**：
1. 在图书列表中选择要归还的图书
2. 点击"还书"按钮
3. 确认归还操作
4. 还书成功后，图书可借数量自动加1

**业务规则**：
- 只能归还当前用户已借阅的图书
- 归还后自动记录归还时间
- 借阅记录状态更新为"已归还"

### 2.5 借阅记录模块

**功能描述**：
- 用户可以查看自己的借阅历史
- 管理员可以查看所有用户的借阅记录
- 显示详细的借阅信息

**操作步骤**：
1. 点击"我的借阅记录"按钮（普通用户）
2. 点击"所有借阅记录"按钮（管理员）
3. 在弹出的对话框中查看借阅记录
4. 点击"关闭"按钮退出

**显示内容**：
- 记录ID、用户名、真实姓名
- 图书名称、作者
- 借阅日期、应还日期、归还日期
- 借阅状态（借阅中、已归还、已逾期）

## 四、操作注意事项

### 4.1 借书注意事项

1. **检查库存**：借书前请确认图书可借数量大于0
2. **借阅天数**：建议设置30天的借阅期限
3. **重复借阅**：同一本书在归还前不能再次借阅
4. **借阅限制**：每个用户最多可借5本书（可在用户表中调整）

### 4.2 还书注意事项

1. **选择图书**：请选择您确实已借阅的图书
2. **确认操作**：点击还书后会弹出确认对话框，请仔细确认
3. **逾期提醒**：如果图书逾期未还，状态会显示为"已逾期"

### 4.3 搜索技巧

1. **支持模糊搜索**：可以输入书名、作者、出版社、分类的部分内容
2. **搜索示例**：
   - 搜索"Java"可以找到所有与Java相关的书籍
   - 搜索"机械工业"可以找到该出版社的所有图书
   - 搜索"计算机"可以找到该分类的所有图书

## 五、常见问题解答

### Q1：忘记密码怎么办？
A：请联系系统管理员重置密码，或直接在数据库users表中修改。

### Q2：借书时提示"您已经借阅了这本书"？
A：说明您已经借阅了该图书且尚未归还，请先归还后再借阅。

### Q3：还书时提示"未找到该图书的借阅记录"？
A：可能是您没有借阅该图书，或者已经归还过了。

### Q4：如何查看借阅期限？
A：点击"我的借阅记录"按钮，可以看到每本书的应还日期。

### Q5：逾期还书有什么影响？
A：当前版本仅标记逾期状态，不影响还书操作。实际应用中可以根据需要增加逾期罚款等功能。
